## API Reference Format

Each API documentation file follows this structure:

1. **Module Overview** - Purpose and main concepts
2. **Classes** - Class documentation with methods
3. **Functions** - Standalone function documentation
4. **Examples** - Usage examples
5. **See Also** - Related modules and documentation

## Contributing to Documentation

When adding new APIs:

1. Document all public classes, methods, and functions
2. Include usage examples
3. Add type hints
4. Document exceptions raised
5. Update this index

See [CONTRIBUTING.md](../../CONTRIBUTING.md) for more details.
